
### 系统分层

```
┌──────────────────────────────────────────┐
│  接入层：Obsidian Plugin (TypeScript)     │  用户交互、diff 预览、审批面板
├──────────────────────────────────────────┤
│  API 层：FastAPI (Python)                │  /workflows/invoke, /resume, /rollback, /pending-approvals
├──────────────────────────────────────────┤
│  编排层：Checkpointed Linear Graph Runner │  ask_graph / ingest_graph / digest_graph
├──────────────────────────────────────────┤
│  服务层                                   │
│  ├─ ask.py (问答主链路)                   │
│  ├─ ingest_proposal.py (治理分析+提案)     │
│  ├─ digest.py (周期复盘)                  │
│  └─ resume_workflow.py (审批恢复)          │
├──────────────────────────────────────────┤
│  能力层                                   │
│  ├─ retrieval/hybrid.py  (FTS5+向量+RRF)  │
│  ├─ context/assembly.py  (上下文装配)      │
│  ├─ tools/registry.py    (工具白名单)      │
│  ├─ guardrails/ask.py    (前后置护栏)      │
│  └─ context/safety.py    (注入检测)        │
├──────────────────────────────────────────┤
│  存储层：SQLite                           │
│  ├─ note / chunk 索引（schema v7）         │
│  ├─ chunk_fts (FTS5) + chunk_embedding    │
│  ├─ workflow_checkpoint                    │
│  ├─ proposal / patch_op / audit_log       │
│  └─ run_trace                             │
├──────────────────────────────────────────┤
│  可观测层                                 │
│  ├─ JSONL trace file                      │
│  └─ SQLite run_trace + CLI 查询           │
└──────────────────────────────────────────┘
```


### 核心模块

|模块|入口文件|职责|
|---|---|---|
|API 路由|`backend/app/main.py`|handler registry 分发 + 统一 outcome builder|
|Ask 问答|`backend/app/services/ask.py`|检索 → 装配 → 工具决策 → guardrail → 生成 → 引用校验|
|Ingest 治理|`backend/app/services/ingest_proposal.py`|scoped note 分析 → 跨 note 证据 → proposal 生成|
|Digest 复盘|`backend/app/services/digest.py`|周期摘要 → proposal 模式可选|
|Graph Runner|`backend/app/graphs/runtime.py`|checkpoint save/load/resume, LangGraph 兼容 shim|
|混合检索|`backend/app/retrieval/hybrid.py`|FTS5 + vector → RRF(k=60) 融合|
|上下文装配|`backend/app/context/assembly.py`|去重 → token 裁剪 → 安全过滤 → ContextBundle|
|工具注册|`backend/app/tools/registry.py`|白名单 + workflow 隔离 + schema 校验|
|护栏|`backend/app/guardrails/ask.py`|tool call 前置 + final response 后置|
|审批恢复|`backend/app/services/resume_workflow.py`|checkpoint 加载 → 状态校验 → writeback → audit|
|Trace|`backend/app/observability/runtime_trace.py`|JSONL + SQLite 双写, compose + 异常隔离|
|插件|`plugin/src/views/KnowledgeStewardView.ts`|diff 预览 + 审批面板 + backend 自启|

### 关键调用链路（以 ask 为例）

```
用户提问
  → POST /workflows/invoke (action_type=ask_qa)
    → handler registry 分发 → invoke_ask_graph()
      → search_hybrid_chunks_in_db()         # FTS5 + vector → RRF 融合
      → build_ask_context_bundle()            # 去重 + token 预算裁剪 + safety 检查
      → _request_tool_call_decision()         # 模型判断是否需要工具
      → evaluate_tool_call_outcome()          # guardrail 前置拦截
      → execute_tool_call()                   # 白名单 + schema 校验后执行
      → _try_generate_grounded_answer()       # cloud → local fallback
      → _validate_generated_answer_citations() # 编号对齐校验
      → evaluate_final_ask_response()         # guardrail 后置拦截
    → checkpoint 持久化 + trace 双写
  → 统一 WorkflowInvokeOutcome → JSON response
```



### 主要技术栈

|层|技术|
|---|---|
|后端|Python 3, FastAPI, Pydantic, SQLite, conda|
|插件|TypeScript, Obsidian Plugin API|
|模型调用|原生 `urllib.request`（非 LangChain SDK）|
|图编排|自研 graph runner，LangGraph 接口兼容 shim|
|检索|SQLite FTS5 + sqlite_vector|
|评测|自研 eval runner + golden case JSON|
