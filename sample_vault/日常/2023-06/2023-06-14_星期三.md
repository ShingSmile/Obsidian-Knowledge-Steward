# 四、今日总结

* 兼容性问题：
	* 流程：
		1. 双写，老接口读老数据，写老数据并开始写新数据  上线1
		2. 刷数据，将老表数据刷到新的数据结构里
		3. 双写，上线新接口读新数据，同时写入老数据和新数据  上线2
		4. 写新数据，读新数据
	* 分析
		* 上线1主要是修改老接口的内容，老版本继续在老接口写入老数据，读老数据，但是老接口也要写新的数据进新的数据结构，新接口同样要写老数据
		* 刷数据，将上线1之前的老数据刷到新的数据结构
		* 上线2才是写好的新逻辑，让版本升级，老接口废弃，新的接口直接读新的数据，但是还要写老数据（很难）
		* 最后直接写新数据，不写老数据  
		* 前两步不是为了过渡刷数据的过程，而是因为一直在运行的老版本一直在产生老数据，不执行第一次上线进行双写，那数据永远不对，除非停服处理
		* 上线2的双写是为了防止有问题的时候还有机会回退到老版本（很难）

* 票面总额 = 销售货款 + 力资开票灌包
* **结算货款**（goodsFullAmount） = 销售货款（saleAmount）- 优惠discount_amount（+零差）
* **应结货款** = 结算货款 - 卖家出三轮车费 + 四个代卖费用(存储在financeinfo中) 
* 支出合计 = 批次结算费用（batch_settlement表） + 代支代付费(agentpay表)
* **结算金额** = 应结货款goodsPaymentAmount+支出合计spendAmount
